HoloSplat — Pack Variants

Drop several .spz, .splat or .ply files that are color/material variants of the same geometry (e.g. cup.left.blue.spz, cup.left.yellow.spz) — same splat count, same positions/scales/rotations, only color/alpha differ. They're grouped by their shared base name and packed into one .spzv file: geometry is stored once, plus a tiny color+alpha palette per variant. The viewer can swap the active variant instantly at runtime with setVariant() — no reload, and only the active variant is rendered.

Independently-trained color variants (each color is its own Gaussian Splatting reconstruction, so splat counts/geometry differ slightly) can't be packed this way — and don't need this tool. Just list the variant suffixes in the object's variants array in the animation JSON (e.g. ["blue", "yellow"] for cup.left.blue.spz / cup.left.yellow.spz) and sync parts in the /holosplat editor — only the active variant is loaded, and setVariant() fetches the others on demand.
